1. Executive Summary

현재 43개 고객사 · 월 매출 6,000만 원 · 직접 원가율 30% 규모는 ERP 초기 설계 목표(약 43개 업체)와 일치하며, 인력 3명(급여 350만 원) 기준으로 실질 순마진 약 36~44%(월 2,100~2,600만 원)로 추정됩니다. 다만 현재 Vercel 데모는 브라우저 저장 방식이라 실무 공동 운영에는 Supabase(DB) 전환이 필수입니다.

핵심 결론

  • 수익성 — 원가 30%는 업종 대비 양호. 성과급·리셀러가 있는 계약은 건당 비용 구조가 달라짐.
  • 인력 — 1인당 약 14~15개 업체 담당으로, 3명 구성은 현재 규모에 적합.
  • 시스템 — ERP 기능·역할 분리는 43개 규모에 맞게 설계됨. DB 연동만 완료하면 실무 투입 가능.
  • 우선 과제 — localStorage 데모 → Supabase 전환, 실제 43개 계약 데이터 이관.

항목 내용

4-2. ERP가 지원하는 실무 흐름

  • 계약 → 입금 확인 → 업무(WorkOrder) → 원가 등록 → 지급 큐 → 성과급 정산
  • 역할별 대시보드: 담당 · 팀장 · 재무 · 임원 · CEO
  • 원가 지급 큐: 은행·계좌 표시, CSV 내보내기
  • 연장 4개월+ 계약 성과급 · 리셀러 10% P&L 반영

4-3. 43개 · 월 6,000만 규모 적합성

구분 판단
기능·규칙 ✅ 43개 규모에 맞게 설계됨 (Supabase 스키마 주석과 일치)
인력 ✅ 3명 · 1인 14~15개 업체 — 적정

4-4. 권장 로드맵

단계 내용 기대 효과
1단계 (즉시) Supabase 연동 · 43개 실데이터 이관 팀 공동 ERP 운영 시작
2단계 원가·성과급·리셀러 건별 입력 정착 P&L과 실제 순마진 일치
