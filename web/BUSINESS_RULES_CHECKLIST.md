## 2. 건당 업무(WorkOrder) 단계 전이

| ID | 규칙 | 기대 동작 | 확인 방법 | 결과 | 메모 |
|----|------|-----------|-----------|------|------|
| W-01 | 자동 생성 | active 계약 선택 시 채널 목표 수만큼 **draft** 건 생성 | `/executions` 계약 전환 → 타임라인 건수 | | |
| W-02 | draft/rejected | **파트너·비용 설정** · 임시 저장 · **업무 배정**(또는 파트너 승인 요청) | draft 건 모달 | | |
| W-03 | 배정/승인 요청 | 기본(`PARTNER_SELF_SERVICE=false`) → **approved** 직행 · 추후 ON 시 **pending_approval** | 업무 배정 후 배지 | | |
| W-04 | 파트너 승인 | *(비활성)* `PARTNER_SELF_SERVICE=true` 시만 → **pending_staff_confirm** | 파트너 역할 승인 | | |
| W-05 | 담당 확인 | *(비활성)* 파트너 승인 확인 → **approved** | staff 확인 버튼 | | |
| W-06 | 결과 제출 | 기본: 담당 **approved**에서 URL/메모 → **delivered** · 추후 파트너 제출 가능 | 실행 진행 결과 등록 | | |
| W-07 | 입금·완료 | staff **입금 확인**(비용>0) 또는 **완료 확인**(비용=0) → **order_ready** | delivered 건 버튼 | | |
| W-08 | 반려 | 파트너/담당 반려 → **rejected** → 다시 W-02부터 | 반려 후 재설정 | | |
| W-09 | 보류·연기 | pending~delivered → on_hold / postponed · **재개** 가능 | 보류·연기·재개 버튼 | | |
| W-10 | 취소 | PAUSABLE 단계에서 cancel → **cancelled** (재개 불가) | 취소 후 단계 | | |

**단계 순서 (정상 경로 · 현재 기본):**  
`draft` → **approved** → `delivered` → `order_ready`  
*(추후 `PARTNER_SELF_SERVICE_WORKFLOW_ENABLED=true` 시)*  
`draft` → `pending_approval` → `pending_staff_confirm` → `approved` → `delivered` → `order_ready`

## 12. DEPLOY_CHECKLIST와 관계

| 문서 | 용도 |
|------|------|
| **BUSINESS_RULES_CHECKLIST** (본 문서) | 규칙이 **맞게 설계·구현**됐는지 (월 1회·대규모 수정 후) |
| **DEPLOY_CHECKLIST** | 배포 전 **5분** 화면 스모크 테스트 |

규칙 검수에서 ❌가 나오면 → 버그 수정 → **본 문서 해당 ID 재검** → DEPLOY_CHECKLIST → Production.
